Mungkin kita pernah sedang mendapatkan project berbasis web dari client dan ingin melihat progress dari web yang sedang kita kerjakan dan ingin mencoba aplikasi yang kita buat secara langsung. Sebenarnya kita bisa saja mengupload hasil kerja kita ke website baik itu ke web server ataupun hosting, namun itu cukup merepotkan dan membutuhkan waktu yang lebih karena harus mengkonfigurasi ini itu.


Ngrok

Namun kerepotan kita diatas akan terjawab oleh sebuah aplikasi yang bernama ngrok. Ngrok adalah sebuah program yang menciptakan tunneling dari internet public (https://sub.ngrok.io) ke port komputer lokal anda. Anda dapat membagikan url yang anda dapatkan ke klien atau ke teman anda sehingga orang lain bisa melihat hasil kerja kita dengan mengakses url yang sudah kita ciptakan menggunakan program ngrok ini tanpa harus repot-repot mengupload hasil project anda ke web server ataupun hosting dengan catatan web server lokal anda dalam keadaan running sehingga kita bisa menghemat waktu.

Pada kasus untuk mencoba ngrok ini, saya menggunakan sistem operasi windows 10 dan XAMPP sebagai package server.

Konfigurasi NGROK

Pertama download ngrok dan pilih sesuai dengan sistem operasi anda. (Download NGROK)

Kemudian ekstrak file ngrok tadi di dalam direktori mana saja. dalam contoh disini, saya menyimpan file ngrok dalam drive D:/Program Files

Setelah itu, masuk ke direktori dimana file ngrok tadi di simpan, dan tekan shift + click kanan dan pilih menu open command window here sehingga membuka command prompt dengan lokasi path direktori kita tadi.

Ketik perintah di bawah pada command prompt untuk melihat perintah-perintah pengoprasiannya.

ngrok http/tcp port

Contoh disini saya menggunakan port 80

ngrok http 80

Maka akan muncul domain forwarding yang ada pada command prompt. Domain forwarding inilah yang akan kita share kepada public untuk mengakses untuk menguji hasil kerja kita yang masih di lokal server.

Program ini sangat membantu sekali, kita tidak harus membuang waktu lagi dengan mengupload dan mengkonfigurasi ke web server atau hosting yang kita miliki hanya untuk pengujian sementara.
